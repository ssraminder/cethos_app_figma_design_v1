# ISO 17100 §3.1 Human Resources — internal audit, 2026-06-12

**Method:** conducted as a Stage 2 surveillance auditor would — live walk of
portal.cethos.com (browser), vendor sample drawn by assignment volume, every
claim cross-checked against the production database. Sample: the three
highest-volume production vendors over 180 days — Abhinav Dang (20
assignments / 24 payables), Gurpreet Singh (9 / 7), Osama Elalwany (7 / 9).

**Scope:** ISO 17100:2015 §3.1 (human resources) incl. Amd.1:2017 §3.1.4.
Production processes (§4–§5) covered separately by
[2026-12-stage2/audit-sample-report.md](2026-12-stage2/audit-sample-report.md).

## Findings

### NC-1 (Major) — §3.1.1/§3.1.4: no qualification records exist for any working vendor

`qms.role_qualifications`, `qms.competence_evidence`, and
`qms.professional_experience` all contained **0 rows** against 1,428 active
vendors, 344 of them assigned work in the last 180 days. All three sampled
vendors show "No role qualifications recorded" on their QMS tab. The
assignment gate fires correctly (warn mode) on every path — the audit trail
is complete — but there is no underlying §3.1.4 evidence for an auditor to
sample. This was already the headline item on the Stage 2 punch-list;
re-confirmed by sampling.

### NC-2 (Major, systemic — **fixed during audit**) — the only qualification-recording mechanism never worked

Every vendor QMS tab fired *"Edge Function returned a non-2xx status code"*.
Root cause: `list-vendor-qms` and `admin-record-qualification` both used
supabase-js clients scoped `{ db: { schema: 'qms' } }`, which PostgREST
rejects (**"Invalid schema: qms"** — the schema is not in the exposed list;
service_role does not bypass this). **"Mark vendor qualified" — the only
human path to record a §3.1.4 qualification — has never worked in
production**, which explains NC-1's zero rows despite the entry UI shipping
on 2026-06-02 (that session verified the modal opened, not that a submit
succeeded).

**Remediation shipped:** PR #940 — public SECURITY DEFINER wrappers
(`qms_list_vendor_qualifications`, `qms_record_qualification_wrapper`),
both edge functions repointed. Read path verified live. Write path deployed;
end-to-end staff verification pending (one *Mark vendor qualified* click on
Test Vendor).

### NC-3 (Minor, partially remediated) — NDA records are split-brain

190 of 344 actively-assigned vendors have legally signed NDAs in the portal
agreements system (`vendor_nda_signatures`, NDA v3.0 live since 2026-06-10),
but the QMS evidence layer (`qms.nda_agreements`) has 0 rows — the QMS tab
told staff "No NDA recorded" for vendors who had signed (all 3 sampled
vendors had current signatures, May 2026). PR #940 surfaces the portal
signature on the QMS tab; the structural bridge (creating `qms.nda_agreements`
rows from portal signatures) lands when the auto-qualification pipeline goes
live.

### NC-4 (Major) — §3.1.7: no documented PM competence

7 active staff (5 admin/PM, 1 reviewer, 2 super-admin); **zero** competence
or training records (`staff_training_progress` empty; no staff path in
`qms.*` — `role_qualifications` is vendor-keyed). §3.1.7 requires documented
PM competence. Small population — an afternoon of data entry once a staff
competence path exists.

### NC-5 (Minor) — §3.1.6: no `reviewer` role in the QMS model

`qms.role_types` holds translator / reviser / post-editor / interpreter.
Review (monolingual, domain-specialist) steps cannot cite a qualification.

### OBS-1 — §3.1.8: maintenance loop unimplemented

`re_qualification_due` exists but nothing sets it; no expiry cron;
`performance_events` never written; CPD evidence type unused; the 160-question
ISO competence quiz bank has **0 submissions ever**. Tooling exists,
evidence layer empty — same pattern as NC-1.

### OBS-2 — §3.1.2: agency sub-contracting

1 active agency vendor. Per-delivery `vendor_identifier` + `business_name`
(shipped 2026-06-04) give traceability of who actually translated. No
qualification route for agencies or their linguists — the auto-qualification
pipeline correctly escalates agencies to a human.

### OBS-3 — §3.1.1 documented process exists but is unapproved

SOP-001 (*How we qualify translators and revisers*) and SOP-002 (*How we
keep qualifications up to date*) are live at /admin/sops in plain language,
fully versioned with DB-enforced immutability after approval — but both are
**drafts**. Activation (which records approver + date) is the §3.1.1 signoff
and also the authorization gate for the pipeline's live mode.

### Positives

- Assignment-eligibility audit trail: 98 % coverage post-launch, every
  assignment path logs a decision (1,963 events / 90 days).
- Reviser ≠ translator separation mechanically enforced — 0 collisions ever,
  1 documented §6.2-style override with written justification.
- Evidence schema models §3.1 nearly clause-for-clause, including
  Amd.1:2017's exact §3.1.4 (a)/(b)/(c) bases.

## Auto-qualification dry run (run `16fee956`, prompt auto-qualify-v2, claude-sonnet-4-6)

All 344 actively-assigned vendors processed (1 extraction error — malformed
JSON from one CV; reprocessable). Nothing was written to
`qms.role_qualifications`; results live in `qms_auto_qualification_results`.

| Decision | Count | Notes |
|---|---:|---|
| **auto_qualify** | **68** | 64× §3.1.4(a) translation degree, 4× §3.1.4(b) degree+2y; avg confidence 0.84; 59 of 68 also reviser-eligible; all 68 have NDAs |
| **escalate** (human queue) | **114** | 77 insufficient evidence, 65 with extractor red flags, 11 CV-vs-declared contradictions, 1 agency, 1 extraction failure |
| **chase** (no CV → automated document request) | **161** | becomes auto/escalate as CVs arrive |

Reading: the moment SOP-001 is activated and live mode is enabled, **68
vendors (~20 % of the active roster) qualify with zero human touch**, each
record carrying a verbatim CV quote, the file's sha256, and our own
payment-history corroboration. The human queue is 114 — larger than the
40–60 estimate because v2 still escalates conservatively (red flags +
sub-0.70 confidence); many will be 30-second confirmations.

**Calibration caveat for the sampling QA:** §3.1.4(c) (5-year experience)
fired **0 times** — the corroboration rule demands our own records back the
claim, and most CVs gave no clean experience timeline. And the top sampled
auto-qualify (Ulf Samuelsson, 0.9) classifies a 1992–94 *translations
course* ("Translations Unit International, Los Angeles") as a translation
degree — Amd.1 requires a degree from a **recognized institution of higher
education**, so non-university training certificates should arguably route
to (b)/(c) or escalate. Phase B should require `level ∈
{bachelor,master,phd}` for §3.1.4(a) and re-run affected rows. This is
exactly the class of issue the monthly 10 % spot-check exists to catch.
